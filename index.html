AI 윤리 핵심가이드

🤔 1. 왜 쓰는지 먼저 생각해요!

AI는 내 생각을 대신하는 게 아니라 돕는 도구예요. 진짜 필요한지 먼저 고민해봐요.

🙋‍♀️ 2. 내 생각을 먼저 말해요!

무작정 묻기 전에, 내 생각과 아이디어를 먼저 정리해보면 훨씬 큰 도움이 돼요.

🕵️‍♂️ 3. AI가 틀릴 수도 있어요!

AI가 하는 말이 항상 정답은 아니에요. 중요한 건 꼭 다시 확인하는 습관을 가져요.

🚀 4. 상상력을 더 크게 키워요!

AI의 대답을 그대로 쓰지 말고, 내 경험과 색깔을 더해서 나만의 멋진 결과를 만들어봐요.

🤫 5. 내 비밀은 지켜요!

이름, 주소, 전화번호 같은 내 정보는 주지 마세요. 힘든 고민은 선생님이나 친구와 나눠요.

🤝 6. 도움받은 건 솔직하게!

AI를 썼다면 숨기지 말고 이야기해요. 정직할 때 나의 진짜 노력이 훨씬 더 빛난답니다.

특수교육 선생님들을 위한 AI 기반 코딩 가이드

바이브 코딩(Vibe Coding)이란?

바이브 코딩은 복잡한 프로그래밍 언어나 문법을 외울 필요 없이, 일상적인 언어(자연어)로 AI에게 명령을 내려 코드를 작성하는 새로운 방식입니다.

선생님들께서 상상하시는 교육용 도구와 아이디어를 대화하듯 텍스트로 입력하면, AI가 이를 실제 동작하는 웹페이지나 애플리케이션으로 만들어줍니다.

특수교육에서 왜 중요한가요?

  • 맞춤형 교보재 신속 제작

    학생마다 필요한 접근성 설정이 다릅니다. 선생님께서 필요한 맞춤형 기능을 AI에게 설명하기만 하면 바로 교보재가 완성됩니다.

  • 접근성(Accessibility) 도구 직접 개발

    시각장애 학생들을 위한 스크린 리더 호환, 고대비, 텍스트 음성 변환(TTS) 등 다양한 UDL 요소가 포함된 도구를 직접 구현할 수 있습니다.

  • 아이디어의 현실화

    코딩 장벽으로 인해 포기했던 혁신적인 교육 아이디어를 이제 기술적인 제약 없이 실현할 수 있습니다.

보편적 학습 설계(UDL) 적용 사례

이 웹페이지 자체도 시각장애인 학생 및 선생님들을 위한 UDL 원칙을 적용하여 만들어졌습니다.

스크린 리더 최적화

시맨틱 태그(header, main, section)와 ARIA 속성을 사용하여 스크린 리더가 화면의 구조를 정확히 읽을 수 있도록 구성했습니다.

키보드 내비게이션

마우스 없이 Tab 키만으로 모든 요소에 접근할 수 있으며, 현재 포커스된 위치가 명확하게 시각적으로 표시됩니다.

고대비 및 가독성

상단의 '고대비 모드 전환' 버튼을 통해 시각적 피로도를 낮추고 텍스트를 더욱 선명하게 읽을 수 있습니다. 폰트 크기 또한 충분히 크게 설정되었습니다.

웹페이지 배포 과정 (GitHub Pages)

작성한 코드를 전 세계 누구나 접속할 수 있도록 인터넷에 배포하는 과정입니다.

  • 1. GitHub에 코드 업로드 (Push)

    작성한 HTML, CSS 코드를 원격 저장소에 업로드합니다. (방금 완료하신 과정입니다.)

  • 2. 저장소 설정 (Settings) 이동

    GitHub 저장소 상단 탭에서 Settings를 클릭한 후, 좌측 메뉴에서 Pages를 선택합니다.

  • 3. 배포 소스 선택

    Build and deployment 섹션에서 Source를 Deploy from a branch로 선택하고, Branch를 main으로 지정한 뒤 Save 버튼을 누릅니다.

  • 4. 배포 완료 및 접속

    약 1~2분 후 새로고침하면 배포된 웹사이트 주소(URL)가 나타나며, 이 링크를 학생들에게 공유하여 바로 사용할 수 있습니다.
